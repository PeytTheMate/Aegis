# Assurance Layer

This project now includes three recruiter-facing assurance capabilities:

1. Requirements-to-evidence traceability
2. Hazard log and safety-case linkage checks
3. Executable formal verification of mode logic

## Inputs

- `assurance/requirements.yaml`
- `assurance/hazard_log.yaml`
- `assurance/safety_case.yaml`
- `formal/mode_logic.tla`

## Outputs

Generated by `python3 -m src.cli assurance check --output-dir assurance`:

- `assurance/traceability_report.json`
- `assurance/traceability_report.md`
- `assurance/safety_report.json`
- `assurance/safety_report.md`
- `assurance/formal_report.json`
- `assurance/formal_report.md`

## CI Gate

The CI workflow runs both:

- unit/integration tests
- assurance check command

If any requirement linkage, safety-case reference, or formal property fails, CI fails.
